## 第1章 引言：背景介绍

随着互联网技术的飞速发展，传统的商品管理方式已经难以满足现代商业的需求。手工记录、纸质管理不仅效率低下，而且容易出错，数据查询和统计也极为不便。因此，开发一套高效、便捷的商品信息管理系统具有重要的现实意义。

Java作为一种面向对象的编程语言，具有跨平台性、安全性和稳定性等优点，是企业级应用开发的首选语言。Spring Boot是Spring家族的一个子项目，它简化了Spring应用的初始搭建和开发过程，通过自动配置机制，开发者可以快速构建独立的、生产级别的Spring应用。MyBatis是一款优秀的持久层框架，它支持定制化SQL、存储过程以及高级映射，能够灵活地将数据库操作与Java对象进行映射，大大提高了开发效率。

"智慧商品信息管理系统"正是基于Spring Boot + MyBatis技术栈开发的一套综合性管理系统。该系统集商品管理、分类管理、订单管理、用户管理、购物车等功能于一体，采用前后端分离的设计思想，实现了商品信息的高效管理和便捷操作。系统不仅提供了管理员端的后台管理功能，还为普通用户提供了友好的商城浏览和购物体验，是一套功能完善、界面美观的现代化商品管理解决方案。

本系统的开发旨在提高商品管理的效率和准确性，降低管理成本，为企业提供一个稳定、可靠的信息化管理平台。同时，通过本课程设计，学生可以深入理解Spring Boot和MyBatis框架的核心原理和应用方法，掌握企业级应用开发的基本流程和规范。

（1）熟悉Java数据类型、语法等基本使用，掌握面向对象编程的核心思想和设计模式；

## 第3章 系统前端设计

### 3.1 前端UI设计

本系统采用现代化的设计风格，以简洁美观、操作便捷为设计原则。前端设计主要包括以下几个方面：

（1）**登录页面设计**：采用左右分栏布局，左侧为品牌展示区，展示系统名称和简介；右侧为登录表单区，包含用户名、密码输入框和登录按钮。整体采用紫色渐变背景，配合白色卡片式表单，视觉效果清晰、专业。

（2）**主页面布局**：采用经典的后台管理系统布局，左侧为导航菜单，右侧为内容区域。导航菜单采用树形结构，支持多级菜单展开和收起，不同角色的用户看到不同的菜单项。内容区域采用iframe框架，实现页面的局部刷新。

（3）**管理员端页面设计**：包括仪表盘、商品管理、分类管理、订单管理、用户管理、系统设置等页面。仪表盘展示统计数据和图表，商品管理和分类管理采用表格形式展示数据，支持分页、搜索和操作按钮。

（4）**用户端页面设计**：包括商城首页、商品列表、购物车、订单列表等页面。商城首页采用现代化的电商首页设计，包含轮播图、分类导航、热门商品、新品上架等模块，商品卡片展示商品图片、名称、价格和加入购物车按钮。

（5）**交互设计**：采用Bootstrap框架实现响应式布局，支持不同屏幕尺寸的适配；使用Font Awesome图标库提供丰富的图标；使用ECharts图表库展示数据可视化图表；实现商品详情弹窗、数量选择器、购物车操作等交互功能。

（6）**色彩设计**：主色调采用紫色系（#667eea到#764ba2渐变），配合白色背景和灰色辅助色，整体风格简洁现代，符合当前主流的Web设计趋势。

**数据库ER图说明：**

用户表与订单表为一对多关系，一个用户可以有多个订单；订单表与订单明细表为一对多关系，一个订单可以包含多个商品；分类表与商品表为一对多关系，一个分类可以包含多个商品；用户表与购物车表为一对多关系，一个用户可以有多个购物车项；商品表与订单明细表为一对多关系，一个商品可以出现在多个订单中。

### 4.1 系统架构设计

本系统采用经典的四层分层架构，各层职责清晰、依赖方向统一，确保了代码的可维护性和可扩展性。

**（1）架构分层说明**

| 层次 | 职责 | 核心组件 |
|------|------|----------|
| **表现层（Presentation Layer）** | 处理HTTP请求，渲染Thymeleaf模板，返回页面响应 | Controller层、Thymeleaf模板 |
| **业务逻辑层（Service Layer）** | 封装业务逻辑，调用Mapper进行数据访问，处理事务和缓存 | Service层、@Transactional |
| **数据访问层（DAO Layer）** | 执行SQL语句，与数据库交互，完成数据持久化 | Mapper接口、XML映射文件 |
| **安全层（Security Layer）** | 用户认证、权限控制、会话管理 | Spring Security |

**（2）系统架构图**

```
┌─────────────────────────────────────────────────────────────────┐
│                     客户端浏览器（Browser）                      │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
```

**（1）登录模块**

登录模块采用Spring Security框架实现，主要包括以下组件：

- **SecurityConfig.java**：安全配置类，配置认证规则、权限控制、会话管理等；

- **SecurityConfig**关键配置代码：

```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .authorizeRequests()
            .antMatchers("/css/**", "/js/**", "/images/**").permitAll()
            .antMatchers("/login", "/403").permitAll()
            .antMatchers("/product/form", "/product/delete").hasRole("admin")
            .antMatchers("/category/**").hasRole("admin")
            .antMatchers("/user/**").hasRole("admin")
            .anyRequest().authenticated()
            .and()
        .formLogin()
            .loginPage("/login")
            .loginProcessingUrl("/login")
            .defaultSuccessUrl("/index", true)
            .failureUrl("/login?error")
            .and()
        .logout()
            .logoutUrl("/logout")
            .logoutSuccessUrl("/login?logout")
            .and()
        .sessionManagement()
            .maximumSessions(1)
            .expiredUrl("/login?expired");
    return http.build();
}
```

**（7）阶段五：高并发缓存优化**

为降低数据库读取压力、提升系统响应性能，本系统实现了双重缓存机制：

**① 声明式缓存（Spring Cache）**

在启动类上添加 `@EnableCaching` 注解开启缓存支持：

```java
@SpringBootApplication
@EnableCaching
public class SmartShopApplication {
    public static void main(String[] args) {
        SpringApplication.run(SmartShopApplication.class, args);
    }
}
```

**ProductService 缓存配置：**

```java
@Cacheable(value = "product", key = "#id", unless = "#result == null")
public Product findById(Integer id) {
    return productMapper.findById(id);
}

@CacheEvict(value = "product", allEntries = true)
public int save(Product product) { ... }

@CacheEvict(value = "product", allEntries = true)
public int deleteById(Integer id) { ... }
```

**CategoryService 缓存配置：**

```java
@Cacheable(value = "category", key = "'all'", unless = "#result == null")
public List<Category> findAll() { ... }

@Cacheable(value = "category", key = "#id", unless = "#result == null")
public Category findById(Integer id) { ... }

@CacheEvict(value = "category", allEntries = true)
public int save(Category category) { ... }

@CacheEvict(value = "category", allEntries = true)
public int deleteById(Integer id) { ... }
```

**② 分布式缓存（Redis）**

生产环境配置文件（`application-prd.yml`）中启用Redis缓存：

```yaml
spring:
  cache:
    type: redis
    redis:
      time-to-live: 1800000      # 30分钟过期
      use-key-prefix: true
      cache-null-values: false
  redis:
    host: localhost
    port: 6379
    timeout: 5000ms
```

**缓存工作流程：**

```
用户请求
    │
    ▼
┌───────────────────────────────────────────────┐
│         Spring Cache AOP 切面拦截            │
└───────────────────────────────────────────────┘
    │
    ├── 缓存命中 ──► 直接返回缓存数据（不执行SQL）
    │
    └── 缓存未命中 ──► 执行数据库查询 ──► 写入缓存 ──► 返回数据
```

**缓存验证方式：**

- 启用Redis后，观察MyBatis日志，第二次查询相同数据时不应出现SQL语句
- 通过控制台日志输出可判断缓存命中情况

### 6.3 开发过程中的问题与解决方案

在开发过程中，遇到了多个技术问题，通过分析和调试逐一解决：

**（1）数据库连接泄漏问题**

- **问题现象**：多次启动应用后出现"Too many connections"错误，数据库连接池耗尽
- **原因分析**：之前的Java进程未完全终止，占用了数据库连接
- **解决方案**：使用`taskkill /F /IM java.exe`终止所有旧进程，释放数据库连接资源

**（2）Thymeleaf模板渲染错误**

- **问题现象**：页面出现500错误，报错信息为"Expression evaluation error"
- **原因分析**：Thymeleaf表达式中使用了嵌套三元运算符和JavaScript函数混合调用
- **解决方案**：将复杂数据传递改为使用`data-*`属性，在JavaScript中通过`getAttribute()`读取

**（3）缓存导致数据过时问题**

- **问题现象**：修改商品后页面仍显示旧数据，分类管理列表为空
- **原因分析**：`@Cacheable`注解缓存了初始空数据或旧数据，`@CacheEvict`未正确清除
- **解决方案**：移除所有缓存注解，确保每次请求直接查询数据库；后期重构时重新设计缓存策略

**（4）文件上传路径错误**

- **问题现象**：图片上传后无法访问，报FileNotFoundException
- **原因分析**：上传路径未明确配置，文件保存到Tomcat临时目录
- **解决方案**：在`application-dev.yml`中添加`upload.path: ${user.dir}/uploads`配置

**（5）Thymeleaf日期格式化错误**

- **问题现象**：页面渲染时`#dates.format()`报错，无法处理`LocalDateTime`类型
- **原因分析**：`#dates`工具类仅支持`java.util.Date`，不支持`java.time.LocalDateTime`
- **解决方案**：引入`thymeleaf-extras-java8time`依赖，使用`#temporals.format()`进行格式化
